    Setup Command


    /setup [log-channel: #log-channel] [mod-rank: @moderator]



    What is the log channel?


    Log channel is required for the bot to function. All Meta Bot logs will be sent into the channel. If the log channel setup is not completed, you cannot use any of the protection modules. Commands are still available.
    We recommend to show Meta's log channel only to trusted server members / staff, since it contains sensitive information such as alt accounts and moderator grief attempts.


    What is the Moderator Rank


    Moderator Rank is the rank, that will be monitored by Meta. Everyone (expect people with Administrator permissions) will be monitored and punished whenever they trigger any of the Anti Mod Grief systems.
    Setting up the Moderator rank is required for the bot to function. If the Moderator Rank is not properly setted up, you cannot use any of the protection modules.

    For example, when a user called "TacoMan" has the rank that is setted as "mod-rank", then Meta will watch TacoMan whenver they act around the server. Moderators can only be punished by the Anti Mod Grief system and only by the events that you have chosen to enable.


    Meta Mute role creation


    Whenever you run the /setup command, Meta will automatically check or create a new role into your server named "Meta Mute", this role is needed for the bot to function.
    This role will be given to those, who trigger Meta protection systems. The role is automatically edited to make the person "not see any" of the server channels when they have it.

    Make sure to check out Meta Mute Setup, because it has to be above the Moderator Rank and below the actual Meta bot role.

    Anti Mod Grief


    /setting [BUTTON]



    Be safe and restrict your server staff to not destroy your server! Meta Anti Mod Grief system protects your server from all kind of griefs, so you better enable it and get to protecting!!



    When Enabled


    Meta will monitor all server members with the Moderator Rank and will make sure they wont damage your server.
    Whenever a Moderator triggers an event, Meta will remove all their roles and will add Meta Mute role to them.
    With Meta Mute role, they will not be able to see/message in any server channel and to any server member.
    Meta will also send a message to the Meta Muted user explaining why they were Meta Muted, and also will send a detailed log to the Log channel.



    Protection Triggers


    Channel Created

    Activated, whenever a user with the Moderator Rank creates any type of channel. It only takes 1 channel to activate the system.
    This module makes it impossible for people with the Moderator Rank to create channels.



    Channel Deleted

    Activated, whenever a user with the Moderator Rank deletes any type of channel. It only takes 1 channel to activate the system.
    This module makes it impossible for people with the Moderator Rank to delete channels.



    Channel Modified Meta Plus

    Activated, whenever a user with the Moderator Rank changes any type of channel's any type of values (permissions, names, etc...).
    This module makes it impossible for people with the Moderator Rank to make changes for channels within the server.



    Manual Ban Remove

    Activated, whenever a user with the Moderator Rank unbans (revokes a ban) via the original Discord Ban Tab.
    It's useful whenever you want that Moderators can only use bots to unban people, because most of the bots will log the event. Yet manually revoking the ban could be to hide a unbanning and could harm the server.



    Dangerous Role

    Activated, whenever a user with the Moderator Rank gives themselves, another person or a bot a dangerous role. We highly recommend to enable this option, because this is one of the main ways servers get destroyed. If your server Moderator gives someone a role that could harm the server, then it posts up a highly dangerous risk!

    When this is enabled, Moderators can still manage roles, but if the role is dangerous, they wont be able to give / remove it from any user.



    Guild Update Meta Plus

    Activated, whenever a user with the Moderator Rank tries to change of the server overview settings (server name, pictures, etc...).

    When a server is being griefed, changing the overview guild settings is very common. Meta protectes against that!!



    Role Created

    Activated, whenever a user with the Moderator Rank creates a new role in the server. This is to stop any attempt to create a new powerful role or mass spam the server with roles.

    When it's enabled, none of the people with Moderator Rank can create roles in the server.



    Role Deleted

    Activated, whenever a user with the Moderator Rank delets a role in the server. This is to stop any attempt to destroy the server by deleting roles or removing roles strategically.

    When it's enabled, none of the people with Moderator Rank can delete roles in the server.



    Role Modified

    Activated, whenever a user with the Moderator Rank changes any role any value in the server. This is to stop any attempt to make a role more powerful or weaker and also to use strategical changes to do harm in the server.

    When it's enabled, none of the people with Moderator Rank can modify roles in the server.






    Anti Alt


    /settings [BUTTON]



    Get notified whenever a possible alt account joins your server. Meta Anti Alt system detects fresh accounts and will notify whenever they join your server.
    As a bonus, you will be able to mute them for a given time whenever you see a reason for that.



    When Enabled


    Meta will monitor all new members who join the server. If someone who joins have a triggering event symptoms, Meta will notify the staff team about that user.
    Also, Meta will send a message to the detected account saying that they have been putted under suspicion.
    Nothing will happen to the account, just a log will be sent to notify the staff.



    Symptoms


    Account Age


    When the account is younger than 2 days, Meta will consider that account to be an Alt account.
    Nothing will happen to the account, just a log will be sent to notify the staff team.






    Malicious Bot


    /settings [BUTTON]



    Don't let a suspicious bot ruin your server. Meta's Malicious Bot detection system will prevent that! Most the the servers get nuked by bots, because it's way faster and easier for the griefers. With this module, only people with Administrator permissions can add bots to your server.



    When Enabled


    Meta will monitor all the bots that get added to the server. If a bot is added to the server and the person who added the bot doesn't have Administrator permissions, Meta will Meta Mute them & the bot that was added.

    Normally, people with Manage Server can add bots, this feature makes sure only Admins can add bots to the server.






    Auto Mod


    /settings [BUTTON]



    Don't let your server get filled with bs! Meta's Auto Mod is designed to be comfortable to use. With one click, you can disable 400+ different racial slurs from your servers. You will also deny all types of discord.gg/ invites / links, to prevent those scammers & spammers.

    Bonus for plus users, is self botting detection!



    When Enabled


    Meta checks all the message contents that are sent in the server. If one of the message contents breaks one of the rules, Meta will delete the message and sends a warning log to the staff Log Channel and to the user.
    Users with Administrator permissions and users with the Moderator Rank are not effected by automod and will not be listened.



    Auto Mod RUles


    Rule #1: Racism


    Meta has blacklisted 400+ different ways to say racial slurs. When an attempt is made, Meta will delete the message and logs it.



    Rule #2: Discord Invites


    Whenever someone sends a message that contains "DISCORD.GG/", Meta will delete the message as this is Discord server invite format.
    After that, a log will be sent to the log channel and to the user.



    Rule #3: Self Botting Meta Plus


    Self Botting is when a user acts like a robot, meaning they are sending embeds, or doing other things that a human cannot do.

    When Meta detects something like this, it will Meta Mute the user and send a message to Staff team.






    Meta / Meta Mute Role



    Meta Mute


    What is that role?


    Meta Mute is a custom role created by Meta setup command.
    That role will be used to make rule breakers / module triggeres not see any server channel and any server members.
    Meta Mute role is required for every single function in the bot.




    IMPORTANT!


    When the role is created, please move it above your staff roles, this is so Meta can prevent Mod Griefs and other server griefs.
    If you do not move it above your Moderation rank or staff ranks, none of the module will work and your server is not protected!

    Also make sure that the Meta Mute role is below the actual Meta role. Otherwise Meta cannot manage that role!




    Meta


    What is that role?


    Meta (role name), is automatically created when adding Meta to your server.
    You cannot delete this role but you can re-name it and modify it all you want.
    Make sure that the role has all the necessary permissions, so it all can work!




    IMPORTANT!


    Please move the role Meta as up as possible. It needs to be higher than Meta Mute role and higher than your Moderation Rank, otherwise the bot cannot protect your server.









    Scan Command



    Scan command is the perfect command to see how protected your server is.
    This command will tell you about your server users, permissions and status of Meta.



    What each field mean in the command?


    Admin Count

    Admin count displays the amount of members in your server, who have Administrator permissions. To not have it as an issue, make sure that your server doesn't have more than 6 people with Administrator permissions.
    Be aware, that this also includes bots!



    Dangerous Roles

    Dangerous roles displays the amount of dangerous roles you have in your server. This is just statistics and doesn't bring down your protection score. To see what a dangerous role is, click here and learn!




    New Accounts

    New account displays the amount of users who are under the suspicioun of being alt accounts. This statistic comes from Anti Alt protection module and when your server has more than 30 new accounts, it will bring down the protection score.




    Server Stats

    Server stats contains the most general server statistics such as the server Member Count, the server Channel Count and the server Role Count. This doesn't affect the protection score at all.




    Protection Score

    Protection score is a ( % ) that will be calculated based on the security level of your server. This is how it's calculated:

    • Log Channel Setup + 5 points
    • Moderator Rank Setup + 5 points
    • Mute Rank Setup+ 5 points

    • Mod Grief Channel Create Enabled + 3 points
    • Mod Grief Channel Delete Enabled + 3 points
    • Mod Grief Chanel Modify Enabled + 6 points
    • Mod Grief Ban Revoke Enabled + 6 points
    • Mod Grief Dangerous Roles Enabled + 8 points
    • Mod Grief Guild Update Enabled + 3 points
    • Mod Grief Role Created Enabled + 3 points
    • Mod Grief Role Deleted Enabled + 3 points
    • Mod Grief Role Modified Enabled + 5 points

    • Anti Alt Protection Enabled+ 12 points
    • Malicious Bot Protection Enabled+ 12 points
    • Auto Mod Protection Enabled+ 4 points

    • Admin Count+ 12 points
    • New Account Count+ 5 points



    Issues:


    Meta will display all your issues and will send a link as well telling how you can fix the issue.
    If you have a perfect score / no issues, Meta will say awesome things to you!





    Inspect Command



    Inspect command is the perfect command to quickly gather information about a specific user.
    This command will tell you about the users basic info, meta info, dangerous info & server info.



    What each field mean in the command?


    Basic:


    User: Clickable mention of the user.
    ID: Users account ID number.
    Joined: A time lenght of when the user joined the server.
    Created: A time lenght of when the user created their account.


    Customization:


    Profile Image: Link to users account profile image.
    Status: Users account status. [Online, Idle, Do Not Disturb, Offline]
    Activity: Users account activity. [Watching, Playing, Streaming, ..., Listening]



    Meta Stats:


    Blacklisted: Does users account contain a blacklisted name.
    Young Account: Is users account younger than 5 days.
    Verified E-mail: Does the user have verified e-mail connected to their account.
    Blocked: Is the user blocked in Meta Database.



    Server:


    Highest Role: Server highest ranking role.
    Is Bot: Whether the user account is bot or no.
    Acknowledgements: Permission wise, what is the user considered as?
    All Roles (Server Name): Display of all users roles.



    Acknowledgements:


    Acknowledgements is devided to 4 different categories. Server Owner, Administrator, Moderator and Server Member.
    Here is what Meta considers each of these:


    Server Owner:


    When user account is assigned to server owner status by Discord.


    Administrator:


    When users account has "ADMINISTRATOR" permissions.


    Moderator:


    When user has at least 1 of the next permissions:
    "BAN_MEMBERS", "KICK_MEMBERS", "MENTION_EVERYONE", "MANAGE_ROLES", "MODERATE_MEMBERS", "MANAGE_GUILD", "MANAGE_MESSAGES"


    Server Member:


    When users account doesn't meet any of the above requirements.







    Dangerous Roles



    Dangerous role is a Discord role that has specific permissions.
    Meta uses the definition of "Dangerous Roles" in Mod Grief module.



    What is considered as a Dangerous Roles?


    Roles that have atleast one of the next permissions:


    • MANAGE_MESSAGES
    • MANAGE_GUILD
    • ADMINISTRATOR
    • BAN_MEMBERS
    • KICK_MEMBERS
    • MENTION_EVERYONE
    • MANAGE_ROLES
    • MODERATE_MEMBERS
