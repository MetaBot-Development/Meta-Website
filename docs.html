    Setup Command


    /setup [log-channel | mod-rank | mute-rank | protection]



    Setup a Log Channel


    /setup [log-channel: #choose-your-log-channel]


    Log channel is required to use the bot. All Meta Protection logs will be sent there.
    It's recommended that only Admins can see that channel.



    Setup a Mod Rank


    /setup [mod-rank: @your-main-moderator-rank]


    Moderator Rank is required to use bot. Without the Mod Rank, features like Anti Mod Grief wont work at all.
    People with the Mod Rank will be monitored by Meta and punished when triggering the Anti Mod Grief System. Learn about Anti Mod Grief



    Setup a Mute Rank


    /setup [mute-rank: create]


    Mute Rank is required to use bot. Without the Mod Rank, none of the features will work.
    When using the command, Meta will create the Meta Ban role that has all the channels configured already.
    That role will be given to protection breakers.



    Setup a Meta Protection Systems


    /setup [protection: mod-grief | anti-alt | safe-bot | auto-mod]


    Enable any of the advanced protection systems that Meta has to offer. To get to know what each module does, check these out:

    • Anti Mod Grief
    • Anti Alt
    • Malicious Bot
    • Auto Mod






    Anti Mod Grief


    /setup [protection: Anti Mod Grief]



    When running that command, you should see a option tab to open up.
    There you can choose the difficulty of your server Moderation Grief System, the more harder, the less freedom your server Moderators have.



    When Enabled


    Meta will monitor all server members with the Moderator Rank and will make sure they wont damage your server.
    Whenever a Moderator triggers an event, Meta will remove all their roles and will add Meta Ban role to them.
    With Meta Ban role, they will not be able to see/message in any server channel and to any server member.
    Meta will also send a message to the Meta Banned user explaining why they were Meta Banned, and also will send a detailed log to the Log channel.



    Difficulties


    Easy


    • Giving Dangerous Roles - User gives themself / someone a dangerous role.
    • Unbanning Manually - User unbans somoene manually, not via bot.



    Medium


    • Giving Dangerous Roles - User gives themself / someone a dangerous role.
    • Unbanning Manually - User unbans somoene manually, not via bot.
    • Creating A Role - User creates a role in the server.
    • Deleting A Role - User deletes a role in the server.
    • Updating A Role - User changes role permissions, color & other values.
    • Server Update - User updates server tab. Meta Plus



    Hard


    • Giving Dangerous Roles - User gives themself / someone a dangerous role.
    • Unbanning Manually - User unbans somoene manually, not via bot.
    • Creating A Role - User creates a role in the server.
    • Deleting A Role - User deletes a role in the server.
    • Updating A Role - User changes role permissions, color & other values.
    • Creating A Channel - User creates a channel in the server.
    • Deleting A Channel - User deletes a channel in the server.
    • Updating A Channel - User updates chnanel. Meta Plus






    Anti Alt


    /setup [protection: Anti Alt]



    When running that command, you should see a option tab to open up.
    You can either enable or disable this feature there, by deafult it's already disabled.



    When Enabled


    Meta will monitor all new members who join the server. If someone who joins have a triggering event symptoms, Meta will notify the staff team about that user.
    Also, Meta will send a message to the detected account saying that they have been putted under suspicion.
    Nothing will happen to the account, just a log will be sent to notify the staff.



    Triggers


    Account Age


    When the account is younger than 5 days, Meta will consider that account to be an Alt account then.
    Nothing will happen to the account, just a log will be sent to notify the staff team.



    Known Alt Name Meta Plus


    When a account joins with a known alt name, Meta will consider that account to be an Alt account. Nothing will happen to the account, just a log will be sent to notify the staff team.






    Malicious Bot


    /setup [protection: Malicious Bot]



    When running that command, you should see a option tab to open up.
    You can either enable or disable this feature there, by deafult it's already disabled.



    When Enabled


    Meta will monitor all the bots that get added to the server. If a bot is added to the server and the person who added the bot doesn't have Administrator permissions, Meta will Meta Ban them & the bot that was added.
    Normally, people with Manage Server can add bots, this feature makes sure only Admins can add bots to the server.






    Auto Mod


    /setup [protection: Auto Mod]



    When running that command, you should see a option tab to open up.
    You can either enable or disable this feature there, by deafult it's already disabled.



    When Enabled


    Meta checks all the message contents that are sent in the server. If one of the message contents breaks one of the rules, Meta will delete the message and sends a warning log to the staff Log Channel and to the user.
    Users with Administrator permissions and users with the Moderator Rank are not effected by automod and will not be listened.



    Triggers


    Rule #1: Racism


    Meta has blacklisted 27 different ways to say racist words. When an attempt is made, Meta will delete the message and logs it.



    Rule #2: Discord Invites


    Whenever someone sends a message that contains "DISCORD.GG/", Meta will delete the message as this is Discord server invite format.
    After that, a log will be sent to the log channel and to the user.



    Rule #3: Self Botting Meta Plus


    Self Botting is when a user acts like a robot, meaning they are sending embeds, or doing other things that a human cannot do.

    When Meta detects something like this, it will Meta Ban the user and send a message to Staff team.






    Meta / Meta Ban Role



    Meta Ban


    What is that role?


    Meta Ban is a custom role created by Meta setup command.
    That role will be used to make rule breakers / module triggeres not see any server channel and any server members.
    Meta Ban role is required for every single function in the bot.



    IMPORTANT!


    When the role is created, please move it above your staff roles, this is so Meta can prevent Mod Griefs and other server griefs.
    If you do not move it above your Moderation rank or staff ranks, none of the module will work and your server is not protected!



    Meta


    What is that role?


    Meta (role name), is automatically created when adding Meta to your server.
    You cannot delete this role but you can re-name it and modify it all you want.
    Make sure that the role has Administrator permissions, so it all can work!



    IMPORTANT!


    Please move the role Meta as up as possible. It needs to be higher than Meta Ban role and higher than your Moderation Rank, otherwise the bot cannot protect your server.








    Scan Command



    Scan command is the perfect command to see how protected your server is.
    This command will tell you about your server users, permissions and status of Meta.



    What each field mean in the command?


    Mod Grief


    Admin Count:

    How many people have Administrator permissions
    This includes BOTS!


    Dangerous Roles:

    How many roles have the status of Dangerous Role


    Anti Alt


    Young Accounts:

    The count of accounts in your server that are younger than 5 days.


    Blacklisted Accounts:

    The count of accounts that have well known raid / spam / troll name.



    Protection Score:


    Protection score is created by the next factors:

    • Have Meta Log Channel setted up +10
    • Have Meta Mod Rank setted up +10
    • Have Meta Ban role setted up +10
    • Have Meta Mod Grief Enabled +10
    • Have Meta Anti Alt Enabled +10
    • Have Meta Malicious Bot Enabled +10
    • Have Meta Auto Mod Enabled +10
    • Have less than 30 new accounts in your server +10
    • Have less than 5 blacklisted named accounts in your server +10
    • Have 5 or less people with Administrator permissions +10


    Issues:


    Meta will display all your issues and will send a link as well telling how you can fix the issue.
    If you have a perfect score / no issues, Meta will say good things to you!





    Inspect Command



    Inspect command is the perfect command to quickly gather information about a specific user.
    This command will tell you about the users basic info, meta info, dangerous info & server info.



    What each field mean in the command?


    Basic:


    User: Clickable mention of the user.
    ID: Users account ID number.
    Joined: A time lenght of when the user joined the server.
    Created: A time lenght of when the user created their account.


    Customization:


    Profile Image: Link to users account profile image.
    Status: Users account status. [Online, Idle, Do Not Disturb, Offline]
    Activity: Users account activity. [Watching, Playing, Streaming, ..., Listening]



    Meta Stats:


    Blacklisted: Does users account contain a blacklisted name.
    Young Account: Is users account younger than 5 days.
    Verified E-mail: Does the user have verified e-mail connected to their account.
    Blocked: Is the user blocked in Meta Database.



    Server:


    Highest Role: Server highest ranking role.
    Is Bot: Whether the user account is bot or no.
    Acknowledgements: Permission wise, what is the user considered as?
    All Roles (Server Name): Display of all users roles.



    Acknowledgements:


    Acknowledgements is devided to 4 different categories. Server Owner, Administrator, Moderator and Server Member.
    Here is what Meta considers each of these:


    Server Owner:


    When user account is assigned to server owner status by Discord.


    Administrator:


    When users account has "ADMINISTRATOR" permissions.


    Moderator:


    When user has at least 1 of the next permissions:
    "BAN_MEMBERS", "KICK_MEMBERS", "MENTION_EVERYONE", "MANAGE_ROLES", "MODERATE_MEMBERS", "MANAGE_GUILD", "MANAGE_MESSAGES"


    Server Member:


    When users account doesn't meet any of the above requirements.







    Dangerous Roles



    Dangerous role is a Discord role that has specific permissions.
    Meta uses the definition of "Dangerous Roles" in Mod Grief module.



    What is considered as a Dangerous Roles?


    Roles that have atleast one of the next permissions:


    • MANAGE_MESSAGES
    • MANAGE_GUILD
    • ADMINISTRATOR
    • BAN_MEMBERS
    • KICK_MEMBERS
    • MENTION_EVERYONE
    • MANAGE_ROLES
    • MODERATE_MEMBERS
