Malicious App Protection

Advanced Anti-Nuke protection module that shields your server from dangerous applications and Discord bots with three powerful protection layers.

The Malicious App Protection System (MBPS) is an advanced Anti-Nuke module that provides three powerful systems to protect your server from Malicious Applications. MBPS works behind the scenes 24/7 and takes action only when specific triggers are pulled by an action.

⚠️
Permission Warning

Users with the Administrator permission will be ignored by Meta. Administrators can change the configurations of the module.

📋
Action Logs

All Malicious App Detector logs will be sent into the server admin-logs channel. You cannot change the logging channel for Malicious App Detector.

Protection Modules

MBPS provides three distinct modules you can enable independently based on your server's needs.

Scanner

Scan Bots

Automatically scans every bot added to your server against the Meta Protection Database of known Malicious Apps.

Restriction

Admin Only

Limits bot-adding permissions to Administrators only, preventing lower-rank staff from introducing threats.

Informative

Notifications

Sends detailed information about any newly added bot and the user who added it to your log channel.

Scan Bots

Database-driven Malicious App detection

Meta Group has gathered — and continues to gather — all known Malicious Apps into the Meta Protection Database (MPD). When this module is enabled, Meta scans every single bot added to your server to ensure it is not in the MPD.

  • If a bot is found in the MPD, it is immediately Meta Muted in the server
  • Server management can remove the Meta Mute role to continue using the bot — at their own risk
  • The scan happens instantly when any bot joins
1 Bot joins server
→
2 MPD scan runs
→
3 Threat detected? Meta Mute applied

Administrator Only

Restrict who can add bots to your server

By default, Discord allows any user with Manage Guild permission to add bots. The Admin Only module restricts this to only users with the Administrator permission.

  • Non-admins who add a bot will be Meta Muted
  • The unauthorized bot will be banned from the server for security
  • Only Administrator permission holders can add new bots
  • Recommended for servers that want tight control over integrations
1 Non-admin adds bot
→
2 Adder gets Meta Muted
→
3 Bot gets banned

Notification System

Stay informed about every bot addition

This is an informative system, not a protection one. Whenever a bot is added to your server, Meta gathers all available information about the bot and the user who added it, then sends a detailed embed to your log channel.

  • Bot info: Name, ID, permissions requested, creation date
  • Adder info: Username, ID, roles, join date
  • Logs are sent to the server admin-logs channel
  • Great for auditing and keeping track of server changes
Privacy Information

The list of known Malicious Apps is hidden and only known by the Meta Team. If you'd like to know if your bot is in the list, please contact the Support team in the Support Server. This information is disclosed on a case-by-case basis.
