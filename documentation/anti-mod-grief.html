Anti Mod Grief Protection

Restrict your server staff members from any of the given functions. When enabled, members with the @Moderator Rank cannot perform the given action.

Anti Mod Grief (AMG) is a background protection system that prevents your server staff from harming your server. Limit your server staff from some more dangerous actions such as deleting a channel/role, unbanning manually without logging, giving out dangerous roles and much more!

⚠️
Permissions Warning

Before using Meta and it's servies, ensure that the app has correct permissions. If you're unsure about the permissions needed, check the permissions documentation or try giving Meta Administrator permissions to test the feature.

Be aware that members with the Administrator permission will bypass all AMG restrictions and can edit the modules without any limitations.

📋
Action Logs

All Anti Mod Grief logs will be sent into the server admin-logs channel. You cannot change the logging channel.

Module Details

Each action can be enabled / disabled independently. Below is a detailed breakdown of what each protection module does and how it works.

Channel Created

Prevents unauthorized channel creation

When enabled, staff members with the Moderator Rank will not be able to create any type of channels in your server. This includes text channels, voice channels, categories, and forum channels.

  • Scope: All channel types are blocked
  • Bonus: Freshly created channel gets deleted instantly
  • Perk: Prevents mass channel creation

Channel Deleted

Blocks channel deletion by staff

Prevents moderators from deleting any channels. This is critical protection against grief attacks where compromised moderator accounts attempt to nuke your server by deleting all channels.

  • Scope: Blocks all channel deletions
  • Recovery: Deleted channels are not recoverable
  • Perk: Channels cannot be deleted, preventing data loss

Channel Modified

Premium feature - Prevents permission changes

Staff members cannot modify channel permissions or settings. This includes changing channel names, topics, slowmode, permission overwrites, and more. This premium feature provides comprehensive protection against stealthy attacks.

  • Scope: All channel settings and permissions
  • Prevents: Channels being taken over by unauthorized users
  • Perk: Smart channel modification detection and logging

Manual Ban Removal

Requires app-based unban

Staff members must use commands to unban users. Direct unbanning from Discord's ban list is blocked. This ensures all unbans are logged and tracked properly.

  • Scope: Discord ban tab unbans being reverted
  • Prevent: Users being secretly unbanned
  • Perk: Complete audit trail of all unbans. Enforces command useage

Dangerous Role

Blocks granting of powerful roles

Prevents staff from assigning roles with dangerous permissions (Administrator, Manage Server, etc.) to members or bots. This stops privilege escalation attacks.

  • Scope: Role and user management
  • Permissions: Dangerous Roles documentation
  • Perk: Both parties (if there is) are neutralized

Guild Update

Premium feature - Protects server settings

Staff members cannot change server overview settings like server name, icon, banner, verification level, and other critical settings. This premium feature prevents defacement attacks.

  • Scope:Server General Settings
  • Prevention: Server defacement, identity theft and impersonation
  • Perk: If possible, setting will be reverted to the previous value

Role Created

Restrict the ability to create new roles

Prevent staff members from creating new roles in the server. This protection ensures that only authorized users can modify the role structure. Make sure no secret powerful roles are being created behind your back.

  • Scope: Roles being created
  • Prevents: Harmful roles being created
  • Perk: Created role will be deleted immediately

Role Deleted

Prevent staff from deleting existing roles

Ensure that your servers role structure is protected from staff members deleting roles. Important and key ranks should be protected from deletion. This module prevents accidental or malicious role deletions.

  • Scope: Roles being deleted
  • Limitation: Deleted roles cannot be restored
  • Perk: Possible data of the deleted role will be preserved

Role Updated

Prevent staff members from modifying existing roles in the server. This protection ensures that only authorized users can modify the role structure. Make sure no roles are being secretly changed or made harmful towards your server.

  • Scope: Roles being modified
  • Limitation: Original states cannot be restored if they were changed
  • Perk: intelligent tracking of role changes

How It Works

1

Action Detected

Meta monitors all server events in real-time using Discord's audit log system.

2

Permission Check

When a protected action occurs, Meta checks if the user has the Moderator rank and isn't an Administrator.

3

Automatic Response

If violation is detected, the action is reversed where possible and logged to admin-logs.
