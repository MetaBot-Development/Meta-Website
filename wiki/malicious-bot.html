What is Malicious Bot Protection System?

Malicious Bot Protection System (MBPS) is an advanced Anti-Nuke protection module, that provides it's users with 3 great systems to protect their server from malicious applications or Discord bots.

The MBPS allows it's user to enable 3 different protection modules:

Scan Bots


Admin Only


Notifications


MBPS works behind the scenes 24/7 and takes action only, when specific triggers have been pulled by an action. To learn more about the system triggers and how they work, continue reading this page!

Permission Warning

Users with the Administrator permissions will be ignored by Meta. Administrators can change the configurations of the module.

Action Logs

All Malicious Bot Detector logs will be sent into the server admin-logs channel. You cannot change the logging channel for Malicious Bot Detector.

Privacy Information

The list of known malicious bots is hidden and only known only by Meta Team. If you'd like to know if your bot is in the list, please contact with the Support team over in the Support Server. This information is disclosed.

How to enable the Malicious Bot Scanner System:

Scan Bots

Meta Group has gathered and will keep gathering all of the known malicious bots into the Meta Protection Database (MPD). If the Scan Bots module has been enabled in a server. Meta will scan every single to bot that is added into the server, to make sure it is not in the MPD.

If an app / bot has been found in the MPD, it will be immediately Meta Muted in the server.
In case the server management wants to continiue using the added app / bot, they can simply remove the Meta Mute role from the application. DO AT YOUR OWN RISK. In most cases, this is not recommended.


Administrator Only

The Admin Only module limites the permission of "Adding applications / Discord bots" into your server. By deafult, Discord allows users to add apps / bots whenever the given user has Manage Guild permission.

With this module enabled, you can limit this permission to only users with the Administrator permissions. This means that even if the user has Manage Guild permission, they cannot add apps / bots into the server.

Whenever someone (who is not an Administrator) adds an app / bot, they will be Meta Muted and the added bot will be Banned from the server for security reasons. We recommended this module, because the permission Administrator is already the most powerful permission, so whoever has it, must be a trusted member of the community.

Adding all kind of different bots could harm your server, so it's best to limit it to the most trusted users in the server.


Notification Option

This module is not a protection system, we consider it as an informative system.

Whenever an app / bot is added into your server, Meta will gather all of the information about the added bot / app and all of the information about the user who added the bot (adder).

All this info will be sent into your server log channel.

If you wish to get an informative embed whenever an app / bot has been added into your server, make sure to enable this feature!
